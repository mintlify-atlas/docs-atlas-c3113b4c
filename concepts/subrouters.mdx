---
title: "Subrouters"
description: "Group routes that share common conditions using nested routers."
---

A subrouter is a `*Router` that is attached to a parent route. The router only tests the subrouter's routes when the parent route's conditions all match. This serves two purposes:

- **Code organisation** — define shared conditions (a host, a path prefix, a scheme) once instead of repeating them on every route.
- **Performance** — if the parent condition fails, the entire group is skipped without testing each individual route.

## Creating a subrouter

Call `Subrouter()` on any route to get a new `*Router` whose routes inherit the parent route's configuration.

```go
r := mux.NewRouter()
s := r.Host("www.example.com").Subrouter()

s.HandleFunc("/products/", ProductsHandler)
s.HandleFunc("/products/{key}", ProductHandler)
s.HandleFunc("/articles/{category}/{id:[0-9]+}", ArticleHandler)
```

The three routes registered on `s` are only evaluated when the request host is `www.example.com`. Requests to any other host bypass this entire group.

## Host-based subrouters

Group all routes for a specific domain under a single host condition:

```go
r := mux.NewRouter()

// API subdomain
api := r.Host("api.example.com").Subrouter()
api.HandleFunc("/users", ListUsersHandler).Methods("GET")
api.HandleFunc("/users/{id}", GetUserHandler).Methods("GET")

// Admin subdomain
admin := r.Host("admin.example.com").Subrouter()
admin.HandleFunc("/dashboard", DashboardHandler).Methods("GET")
```

## Path-prefix subrouters

When a subrouter's parent route uses `PathPrefix`, inner routes use that prefix as their base path — you only write the remainder.

```go
r := mux.NewRouter()
s := r.PathPrefix("/products").Subrouter()

// Handles /products/
s.HandleFunc("/", ProductsHandler)
// Handles /products/{key}/
s.HandleFunc("/{key}/", ProductHandler)
// Handles /products/{key}/details
s.HandleFunc("/{key}/details", ProductDetailsHandler)
```

The subrouter prepends `/products` to each inner path automatically.

## Combining conditions

You can combine any number of matcher methods before calling `Subrouter()`:

```go
r := mux.NewRouter()

// Only HTTPS requests to api.example.com under /v2
v2 := r.Host("api.example.com").
    Schemes("https").
    PathPrefix("/v2").
    Subrouter()

v2.HandleFunc("/widgets", WidgetsHandler).Methods("GET")
v2.HandleFunc("/widgets/{id}", WidgetHandler).Methods("GET", "PUT")
```

## Nested subrouters

Subrouters can themselves have subrouters:

```go
r := mux.NewRouter()

// All routes under /api
api := r.PathPrefix("/api").Subrouter()

// All /api/v1 routes
v1 := api.PathPrefix("/v1").Subrouter()
v1.HandleFunc("/users", ListUsersHandlerV1).Methods("GET")

// All /api/v2 routes
v2 := api.PathPrefix("/v2").Subrouter()
v2.HandleFunc("/users", ListUsersHandlerV2).Methods("GET")
```

A request to `GET /api/v2/users` passes the `/api` check, then the `/api/v2` check, and finally matches the `/users` handler.

## Named routes in subrouters

Routes registered on a subrouter can be named and used for URL building. The subrouter shares the named route registry with the root router, so you look up routes by name on the root router regardless of which subrouter they were registered on.

```go
r := mux.NewRouter()
s := r.Host("{subdomain:[a-z]+}.example.com").Subrouter()

s.Path("/articles/{category}/{id:[0-9]+}").
    HandlerFunc(ArticleHandler).
    Name("article")

// Build the full URL including host and path
url, err := r.Get("article").URL(
    "subdomain", "news",
    "category", "technology",
    "id", "42",
)
// http://news.example.com/articles/technology/42
```

When the subrouter inherits a path prefix, that prefix is automatically included when building URLs:

```go
r := mux.NewRouter()
s := r.PathPrefix("/products").Subrouter()

s.HandleFunc("/{key}/details", ProductDetailsHandler).Name("product-details")

url, err := r.Get("product-details").URL("key", "widget")
// /products/widget/details
```
