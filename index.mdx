gorilla/mux is the most popular HTTP router for Go. It implements the `http.Handler` interface, making it a drop-in replacement for Go's standard `http.ServeMux` — with far more expressive route matching, named URL variables, subrouters, middleware, and reverse URL building.

## Why gorilla/mux?

The standard library's `http.ServeMux` only matches on URL paths. gorilla/mux adds:

- **Named path variables** — capture `{id}` or `{id:[0-9]+}` from URLs
- **Multi-condition matching** — combine path, host, HTTP method, scheme, headers, and query params
- **Subrouters** — group routes that share common conditions for cleaner code and faster matching
- **Reverse URL building** — generate URLs for named routes programmatically
- **Middleware** — wrap handlers with reusable logic using the standard `func(http.Handler) http.Handler` signature

## Get started

<Steps>
  <Step title="Install the package">
    ```bash
    go get -u github.com/gorilla/mux
    ```
  </Step>
  <Step title="Create a router and register routes">
    ```go main.go
    package main

    import (
        "fmt"
        "net/http"
        "log"

        "github.com/gorilla/mux"
    )

    func main() {
        r := mux.NewRouter()
        r.HandleFunc("/", HomeHandler)
        r.HandleFunc("/products/{id}", ProductHandler).Methods("GET")
        log.Fatal(http.ListenAndServe(":8080", r))
    }

    func HomeHandler(w http.ResponseWriter, r *http.Request) {
        fmt.Fprintln(w, "Welcome!")
    }

    func ProductHandler(w http.ResponseWriter, r *http.Request) {
        vars := mux.Vars(r)
        fmt.Fprintf(w, "Product: %s\n", vars["id"])
    }
    ```
  </Step>
  <Step title="Run your server">
    ```bash
    go run main.go
    ```
    Your server is now running at `http://localhost:8080`.
  </Step>
</Steps>
